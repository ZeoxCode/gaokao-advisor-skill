# 位次换算引擎（skill 自己算，不再外包）

核心原则：家长懒得用工具才来问 AI。所以 AI 必须能亲自完成位次换算，只要家长能提供原始数据。绝不把"去用考试院系统"当作答案推回去。

## 总纲：联网强模型 = 现场数据库，但必须自带核验刹车

如果执行本技能的是带联网搜索的强模型（Gemini/GPT等），它就是一个实时录取数据库——不需要预建库，自己现场把一分一段表、各校各专业往年录取位次、招生计划全查出来、现场换算。**放手让它查全、查到专业颗粒度。**

但"能查"不等于"查得准"。强模型最危险的失误模式是：给出一个格式工整、来源像样、但实际错了的数字（分数线当成位次、抓错年份、相近数据拼接），而家长完全看不出来。报志愿的滑档往往就死在这种"看着对、错几百名"的数字上。所以放权的同时，强制套上六条核验刹车：

1. **每个关键数字必须标注"年份 + 来源 + 是分数还是位次"**。例：「南开大学历史类某组 2024年 录取最低位次 约X名（来源：江苏省考试院2024投档线）」。含糊的数字不许进表。
2. **交叉验证**：每个用于定档的关键位次，至少用两个独立来源比对（考试院官方 + 报考网站 / 学校招生网）。两个来源对不上，取保守值并标注分歧。
3. **分数与位次绝不混用**：看到的是分数就明说是分数，必须经一分一段表换算成位次后才能比较。严禁拿分数直接当位次用。
4. **新老高考、改革年单独警示**：跨改革年的数据参考性下降，明确标注"该年为改革年，往年位次仅供参考"。
5. **不确定就标"待核"，不许编**：查不到权威数据的，宁可写"此项需家长用考试院系统核实，建议问法：…"，绝不用一个估算的数字填进表里冒充确定。
6. **单一来源数字禁止下绝对结论**：基于没经交叉验证的数字，不许说"稳进""绝对安全""接近100%"，一律降级为"参考区间，需核实"。

**一句话：让强模型当数据库，但每个数字都要它自己交叉验证、标年份来源、不确定就认。给油门，也给刹车。**

**核验刹车不只管位次——管所有数字**：位次、招生名额、就业率、升学率、薪资、保研率、学科排名、男女比例……凡是报告里出现的具体数字，都适用上面六条刹车（交叉验证、标年份来源、不确定标"待核"、单一未核来源不下绝对结论）。专业核实和学校体检里大量"凭印象写的数字"（如"统计全国前三""升学率65-75%""某私募偏爱本校"）尤其危险——模型容易凭训练记忆编一个看着对的数。这些必须现场核实或标"待核"，绝不当确定结论给家长。

## 一、AI 需要哪些原始数据（主动找齐，能搜的自己搜）

换算需要三样，按优先级获取：
1. **孩子的分数 + 今年本省一分一段表**（把分数→今年位次）。一分一段表每年出分后省考试院发布，AI 应先尝试联网搜"〔省份〕〔年份〕一分一段表"；搜不到就请家长把表（或孩子分数对应的位次行）贴进来——表是公开数据，家长截图即可。
2. **目标校往年录取数据**：往年"投档线分数"或"录取最低位次"。若给的是分数，需要**那一年**的一分一段表来翻译成位次。AI 联网搜"〔学校〕〔省份〕〔往年〕录取分数线 位次"，多数报考网站直接给了往年位次，能省去翻历史一分一段表。
3. **本省招生计划/批次规则**：平行志愿数量、是否专业组模式。

## 二、换算步骤（AI 照此心算/列式，不依赖外部工具）

```
第1步 定孩子位次：
  孩子分数 → 查今年一分一段表 → 今年全省位次 R0
  （一分一段表是"该分数及以上累计人数"，R0≈该分数对应的累计排名）

第2步 定每所目标校的"门槛位次"：
  优先直接用报考网站给出的"该校去年录取最低位次" Rschool
  若只有分数：用该校往年投档线分数 → 查那一年的一分一段表 → 换成那年位次

第3步 比较（关键：比位次，不比分数）：
  R0 < Rschool（孩子排名更靠前）→ 该校相对稳/保
  R0 ≈ Rschool                    → 稳
  R0 > Rschool（孩子排名靠后）    → 冲

第4步 看趋势不看单点：
  取该校近3-5年录取位次，看波动方向和大小年规律，
  用区间而非单一数字判断（如"近三年在8000-11000名波动，孩子9500，
  属于稳但遇大年偏险"）。
```

## 三、把位次差翻译成人话的概率档（给家长直觉）

不输出虚假的精确百分比（AI 算不出真实录取概率），而用经验档位，并说明依据：
- 孩子位次比该校门槛**靠前 20% 以上** → "保底级，正常发挥很稳"
- 靠前 5%-20% → "稳，主力填报区"
- ±5% 以内 → "临界，可冲可稳，受大小年影响大"
- 落后 5%-15% → "冲，有机会但需运气"
- 落后 15% 以上 → "够不太着，除非今年爆冷，不建议占名额"

并永远附一句：以上基于往年位次推演，今年实际录取线取决于当年报考人数，仅供分层参考，非录取保证。

## 四、AI 必须做的诚实声明（防止算错害人）

- 每次换算明确说明用的是哪一年的一分一段表、数据来源。
- 改革年（选科调整/批次合并/专业组重组）明确警示往年位次参考性下降。
- 拿不到权威一分一段表时，宁可说"请把这张表贴给我，我马上算"，也不靠记忆编造位次——**位次编错直接报志愿翻车，这是底线**。
- 新高考"专业组"模式下，门槛位次要落到"组"而非"校"，不同组位次差异大。

## 五、一句话定位
skill 的位次能力 = **联网强模型现场当数据库自己查全（不预建库、查到专业颗粒度）+ 六条核验刹车（每个数字交叉验证、标年份来源、分数位次不混、改革年警示、不确定标待核、单一来源不下绝对结论）+ 会读趋势（看区间和大小年）**。给油门也给刹车——既不缩手缩脚把活推给家长，也不放任它用查错的数字坑人。

## 六、预估也要出，不卡家长（铁律）

**绝不以"今年正式数据未公布"为由拒绝服务或只给半成品。** 出分前的预估期恰是家长最焦虑、最需要规划的时候。只要有任一形式的位次依据，就正常走完整流程、出完整结果：

| 输入精度 | 处理方式 | 标注 |
|---|---|---|
| 正式位次（出分后） | 正常精算 | 无 |
| 估分（出分前自估） | 用最近1-2年一分一段表把估分换算成参考位次，按参考位次出完整结果 | "基于估分，出分后必复核" |
| 模考/平时排名 | 取模考全省/全市位次当锚点（提示：模考范围≠高考全省，有偏差），给区间版完整规划 | "基于模考、波动较大，方向参考" |

- **预估版照样完整**：照样填满志愿表、照样冲稳保、照样能接体检。全程标注精度等级，风险总账置顶"正式位次公布后请回来复核"。
- **预估时主动放宽缓冲**：估分有误差，冲稳保梯度比正式数据时拉得更宽——保底压更低、多备几所深兜底，抵御估分偏差滑档。

## 七、招生名额查当年：出了用今年，没出按历年估（铁律）

位次是孩子的竞争力，**名额是学校今年给本省留的坑位**——坑位每年变，去年的位次参考价值取决于今年坑位变没变。出志愿表前必做：

1. **主动联网查今年名额是否公示**：搜"〔学校〕〔年份〕〔省份〕招生计划/分省分专业计划"。权威源：阳光高考平台、本省考试院、学校招生网。
2. **已公示** → 用今年真实名额，并必须对比去年：
   - **扩招**（名额增多）→ 录取位次通常松动，往年位次可适当放宽参考；
   - **缩招**（名额减少）→ 往年位次会变紧，**滑档的隐形杀手**——孩子按去年位次以为稳，今年缩招可能够不着。缓冲拉大并写进风险提示。
3. **未公示**（出分前常见）→ 用近1-2年名额预估，标注"今年名额未公布，基于历年估算，公示后需复核；若今年扩/缩招，结论会变"，并列入风险总账。
4. **位次+名额联动原则**：两个都用当年数据；都没有就都按历年预估、都标待核——但**都照样出完整结果**，不卡家长。

## 八、用户贴数据时的处理纪律（家长粘贴表格/截图/志愿卡结果时）

家长常会把现成数据贴进来：省考试院的一分一段表、本省投档线、招生计划、学校招生网的分省分专业计划，或掌上高考/优志愿/夸克/蝶变/志愿卡导出的候选表、截图、PDF、复制的文本。此时**既不要盲信、也不要无视从零重搜**，按下面的纪律处理：

1. **先判来源、再定可信度（分级）**：
   - 官方源（省考试院 > 学校招生网/阳光高考）= 最高可信，可直接用。
   - 第三方工具（掌上高考/优志愿/夸克/蝶变/各类志愿卡）= 只当**候选池**，关键数字（位次、招生人数、投档线）要回官方源复核。
   - 家长手填/截图/记忆 = 最低可信，逐项核。
2. **不盲信**：第三方志愿卡也会错、截图可能是往年的、家长可能记错——用户贴的数据一律先核可信度，不当成确定事实。
3. **不浪费**：用户已经贴了候选池，就**在它基础上做粗筛/冲稳保/查漏**，不要假装没看见、从零重新搜一遍（既慢又可能更差）。
4. **抽取关键字段**：从贴进来的数据里提取——学校、专业组、专业、年份、最低分、最低位次、招生人数、学费、校区，整理成可计算的候选池。
5. **标注状态**：每条标「已核（官方源）/ 单源待核 / 第三方待复核 / 缺字段」，让家长一眼看清哪些可信、哪些要再查。
6. **缺字段就补查或标缺**：贴进来的数据缺了关键列（如只有分数没位次），按本手册前述方法换算补全，补不了就标「缺，需核」，不编。
7. **官方优先级覆盖冲突**：若用户贴的第三方数据与官方源对不上，以官方为准并提示分歧。

**一句话**：用户贴的数据是"省力的起点"，不是"免核的终点"——接住它、分级它、复核关键数字，再进入冲稳保流程。
