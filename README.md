# 高考志愿全程顾问 · gaokao-advisor

> 站在「孩子 + 家长」一边的高考志愿决策助手。
> 别人帮你算「能上哪」，它帮你判断「该不该去、去了会怎样、这笔账划不划算」。

`gaokao-advisor` 是一个 [Agent Skill](https://docs.claude.com)，为支持 Skill 机制的 AI 助手（Claude、以及可加载工作手册的 GPT / Gemini 等）提供一套完整的高考志愿决策工作流。它的定位不是市面上常见的「算分匹配器」，而是一个**只对家长+孩子说真话、现实**的参谋。

## 它解决什么问题

市面上的 AI 志愿工具大多是**算分匹配器**：输入分数和简单的基本信息，输出「你能上哪些学校」。它们的核心是录取数据库，立场上服务于平台与机构（本质上是一个促进成交的中介）——所以它们只告诉你「能上」，不告诉你「该不该去、去了会不会被坑」。

本技能的立场只有一个：**站在孩子和家庭这一边**。招生简章、官方排名、学校宣传里的好话，不当作结论。它专门回答家长+孩子关心、但是ai志愿卡、报志愿工具没有讲的问题：

### 底线安全（孩子交出去，学校是不是真的关心学生）

- 历年有没有出现过学生非正常死亡、极端事件、严重维权或异常高的退学延毕——这类「损耗」很多官方材料不会主动呈现，得去舆情和存档里扒。
- 真出事时，学校是公开通报、担责、推动整改，还是沉默、压舆情、让家属和学生自己消化？这比「出没出过事」更能看清它会怎么对你孩子（相关说法一律标注是否经证实、可信度多高）。
- 女孩家庭：有没有偷拍、性骚扰、师生权力不对等的公开线索？若出现类似事件，学校处理的是加害者，还是让受害者承担额外代价？
- 新校区是不是偏远、离医院多远、心理咨询是真有人还是摆设、辅导员和危机响应能不能真的保护孩子。

### 校风与日子（孩子进去以后，四年到底怎么过）

- 官不官僚、办事折不折腾、评奖评优透不透明、学生会和辅导员生态压不压人。
- 本科阶段主要看辅导员、任课老师、学院行政和评奖保研生态；若未来可能本校读研，再重点核查导师 PUA、压榨、抢成果等风险线索，以及受委屈时有没有正常申诉通道。
- 孩子每天的真实生活：宿舍几人间有没有独卫空调、食堂贵不贵、哪几门课最容易挂、哪些老师给分友好、选课和转专业是不是真能操作、几点断电。

### 专业成色（名字好听，不代表值得读）

- 这个专业在这所学校，是重点培养的亲儿子，还是招生目录里凑数的边缘专业？
- 有没有好老师、好实验室、保研名额、竞赛资源、对口企业，还是只有一个漂亮的专业名。
- 给分压不压、课程难不难，会不会影响保研、出国 GPA 和后续升学。

### 生源圈层（孩子和谁同窗四年）

- 本地生源多不多——去外地怕融不进去、在本地图人脉，策略正好相反。
- 同学的消费水平、城乡结构、家庭背景对不对得上，孩子会不会明显不适应。
- 男女比例、校区位置、城市氛围——这些实实在在影响校园社交和生活体验，也是部分家庭会纳入的现实考量。

### 真实出路（牌子背后的通道是什么）

- 这学校的行业血统是什么？是不是原部委、行业系统、地方重点单位真正认可的学校（名字朴素但出身硬、被对口系统长期认可的，正是这类）。
- 毕业生到底去了哪些单位——不是「就业率很高」，而是具体到公司名、岗位、地域、薪资区间和升学去向。
- 有没有外人不知道、但稳定、有含金量、长期专招这几所学校的研究所、国企、央企、事业单位。

### 家庭现实（普通家庭不能只听宣传）

- 家里在体制内、医疗、电网、金融等行业/系统有资源，哪些专业能把资源变成孩子的真实优势。
- 没人脉的普通家庭，哪些方向看着光鲜、其实高度依赖关系和长期托底，别把高分和家庭投入押到低回报路径上。
- 经济下行期，哪些选择更抗跌、更容易上岸、更不容易被行业周期吞掉——铁饭碗、牌照、编制的权重，高于「风口」和「高起薪」。

`gaokao-advisor` 专门挖这些「招生简章不会写、普通志愿卡不愿碰、但决定孩子四年和未来」的东西。它不替你做决定，—只把能查的查清楚，把风险、代价和机会摆清楚，最后把决定权交还给家庭。

## 核心特点

| 维度 | 说明 |
| --- | --- |
| **立场：只为孩子和家长** | 与招生方、学校、平台天然对立。一切官方口径默认反过来审视，不为任何学校的声誉负责，只为这一个孩子的安全和前途负责。 |
| **深度：挖别人不敢碰的真相** | 退学率、极端事件密度、师生权力生态、用保研名额私下摆平事故、给分压不压、分配看不看关系——这些是商业产品因风险而回避的部分。 |
| **颗粒度：细到可执行** | 不停在「就业不错」「校风良好」这类正确的废话。落到「哪门课是挂科分水岭」「哪个校区有没有独立浴室」「这专业本科直接就业的人去了哪、拿多少」。 |
| **现实：敢算功利账** | 行业血统、第一学历的终身烙印、城市的阶层浓度、家庭资源如何变现、甚至婚恋市场的现实考量——家长私下会算的账，照实给，不做道德评判。 |
| **诚实：不编、不装懂** | 查不到的数据老实说查不到、只给定性判断；薪资给分布不给天花板；任何未经核实的数字一律标注「待核」。 |
| **克制：不替你拍板** | 把选项、利弊、代价、矛盾摆到最清楚，最终决定永远交还家庭。连「逆孩子意愿」这种选择，也只做一次「风险定价」，不说教。 |

## 工作流概览

整套决策分四个阶段，像导航一样牵着家长一步步走，每个阶段收尾都给出「岔路口选项 + 默认推荐」，等家长选择后再继续：

```
① 报志愿        从分数/位次出发，做「城市 · 学校 · 专业 · 就业」四支柱决策，
                产出一张「填满整张表」的完整候选名单（按各省真实志愿数）

      ↓  （能上 ≠ 该去，引导深挖）

② 学校体检      对在意的几所学校做八维深度核查：
                报考技术 / 底线安全 / 行业血统 / 生源画像 /
                校风治理 / 学业出路 / 城市生活 / 暗规则

      ↓  （定下学校后，核实要报的专业）

③ 专业核实      两层穿透：「专业本身好不好」×「这校的这专业什么成色」
                方向分化 / 课程颗粒度 / 真实就业出口 / 绩点经济学

      ↓

④ 最终排表      成品自检 → 一页纸结论 → 风险总账 → 待复核提醒
```

家长可以从任意阶段进入（只有分数 → 阶段一；直接问「某大学怎么样」→ 阶段二），流程为人服务，不强制顺序。

## 设计原则（写进 Skill 的铁律）

这些原则约束 AI 的每一步输出，是整个项目的灵魂：

1. **不决策，只呈现** — 职责到「把选项利弊摆到最清楚」为止，决定永远是家庭的。模糊诉求点破、矛盾诉求摊开、重大人生选择（复读/出国）只算账不下「一票否决」。
2. **站孩子 + 家长这边，和机构天然对立** — 彻底清洗官方/招生/学校视角。
3. **挖到底、不偷懒** — 能查的自己查透，不把活甩回给家长（如「建议您去问招生办」是失败做法）。
4. **给油门也给刹车** — 数据让联网模型现场查全，但每个数字都要交叉验证、标注年份来源、不确定就标「待核」，绝不靠记忆编造。
5. **真实第一，编造零容忍** — 负面信息挖到底，但传言标传言、不夸大、不杜撰。
6. **安全类风险厌恶优先** — 涉及孩子安全的未证实线索，按「值得警惕」对待而非「证据不足」打折。
7. **按经济下行期算** — 不带「风口有前途」的盛世幻想，核心是帮孩子上岸、饿不死、裁不掉、贬值慢。

## 推荐运行顺序

**优先 Gemini，其次 OpenAI，再次 Claude。**

本技能是一个「现场全网深挖」的调查工具，不内置录取数据库，而是依赖执行模型实时检索、交叉验证、挖掘舆情与存档。谁的搜索覆盖更广、中文互联网挖掘能力更强，最终报告就越接近家长真正需要的"可决策情报"。

1. **首选 · Gemini 系列**：推荐在 **Antigravity + Gemini** 中运行，或在 **Google AI Studio** 中作为工作手册使用。
2. **次选 · OpenAI GPT 系列**：推荐放到 **Codex** 中使用，搜索和推理能力扎实。
3. **再次 · Claude**：适合用 `.skill` 包快速上传体验，推理和遵循手册能力强，但中文舆情检索覆盖相对依赖工具环境。

经验法则：**哪个平台的模型搜得更广、更深，本技能就跑得越好。** 弱搜索或离线环境下，它会退化为「只能给方法、给不出具体情报」。

> 各家模型版本更新很快，本文不写死具体版本号；同一平台请优先选择当前可用的最强推理 + 联网检索组合。

## 安装与使用

### 方式一：作为 Agent Skill 安装（推荐：Antigravity / Codex）

本仓库中的 `gaokao-advisor/` 目录已经是标准 Agent Skill 结构，可直接放入支持 Skills 的工具中。

**全局安装（Antigravity 桌面版 / Codex 通用）：**

```bash
mkdir -p ~/.agents/skills
cp -R gaokao-advisor ~/.agents/skills/
```

**仅当前项目安装：**

```bash
mkdir -p .agents/skills
cp -R gaokao-advisor .agents/skills/
```

**Antigravity CLI 全局安装：**

```bash
mkdir -p ~/.gemini/antigravity-cli/skills
cp -R gaokao-advisor ~/.gemini/antigravity-cli/skills/
```

安装后新建对话，询问"有哪些 skills 可用？"确认是否已识别到 `gaokao-advisor`。

### 方式二：作为 Claude Skill 安装

1. 下载本仓库的 `gaokao-advisor.skill` 文件。
2. 在 Claude（claude.ai 或桌面 / 移动应用）的 **设置 → 技能（Skills）** 中上传该文件。
3. 新建对话，用自然语言描述需求即可，例如：

   > 帮我看看孩子报志愿。河北考生，女孩，物化生，估分 600 左右，想去一线城市、至少读硕士。

   AI 会自动调用本技能，从位次换算一路带到填满志愿表，再引导你深挖心仪的学校和专业。

> 对 Claude 用户：直接上传 `gaokao-advisor.skill` 即可使用，不需要单独下载 `SKILL.md` 或 `references/` 里的 Markdown 文件；这些参考手册已经打包在 `.skill` 文件中。本技能不内置录取数据库，实际效果取决于 Claude 当前是否具备联网检索能力。若无法联网，它只能提供方法和分析框架，不能保证查到最新学校、专业和位次信息。
>
> Skills 功能需要相应的订阅版本。若设置中无此入口，请确认你的账户类型。

### 方式三：作为工作手册供其他 AI 使用

本技能本质是一套结构化的 Markdown 工作手册，不依赖特定平台。你可以将 `SKILL.md` 与 `references/` 目录下的手册内容提供给支持长上下文、且具备联网搜索能力的 AI，让其据此执行同样的工作流。

## 它能做什么 / 不能做什么

**它擅长的（判断与调查）：**
- 把一所学校挖到底，呈现招生宣传之外的真实面貌。
- 把一个专业核实到方向级、课程级、真实出口级。
- 在多个候选间做结构化对比，并结合你家情况给出有主次的判断。
- 把功利、现实、敏感但有用的维度照实摆出来当决策弹药。

**它的边界（必须诚实告知）：**
- **数据准确性依赖执行模型的实时检索能力。** 它不内置录取数据库，而是引导联网模型现场查证并交叉验证。涉及精确位次换算时，强烈建议结合省考试院官方数据或专业志愿工具的导出数据使用——**位次算错会导致滑档，这是底线**。
- **它给判断，不给保证。** 所有结论基于公开信息，可能不完整或过时，最终决策与核实责任在使用者。
- **最佳用法是「相互印证」**：用省考试院系统或专业志愿工具核准「能上哪」，再用本技能深挖「该不该去」——包括学校安全、校风治理、专业成色、真实就业、城市生活和待复核风险。两边互相校验，决策更稳。
